📚 Complete Documentation

🚀 Quick Start Guide

1

Prepare Your Layers

Create PNG layers for each trait (background, body, eyes, etc.). Organize them in folders with prefixes like 00_Background, 01_Body, 02_Head, etc.

2

Upload Individual Layer Folders

Upload each layer folder separately (e.g., 00_Background, 01_Body, 02_Head). Do NOT upload the parent folder containing all layers.

3

Configure Active Layers

Toggle layers on/off and set probability percentages. For example: 100% for backgrounds, 50% for gear, 20% for accessories.

4

Generate & Monitor Progress

Click generate and watch the live progress modal showing "X of Y NFTs generated" with real-time updates.

📁 Layer Organization & Smart Trait Detection

🎯 Automatic Trait Detection

The system automatically detects trait categories from your folder structure and assigns proper layer orders:

  • Background (Order 1) - Base backgrounds for all NFTs
  • Base (Order 2) - Main character/base layer
  • Body (Order 3) - Body variations
  • Head (Order 4) - Head/face variations
  • Eyes (Order 5) - Eye expressions
  • Mouth (Order 6) - Mouth expressions
  • Accessories (Order 7) - Hats, glasses, etc.
  • Gear (Order 8) - Clothing, armor, etc.

🖼️ Image Processing & Optimization

⚡ Automatic Compression

Large images are automatically compressed to 1024x1024 pixels for optimal performance:

  • Maintains aspect ratio during compression
  • Sequential processing to avoid memory issues
  • Fallback to original images if compression fails
  • Significantly improves generation speed

💾 Memory Management

Built-in optimizations for efficient resource usage:

  • Low memory mode enabled by default
  • Automatic garbage collection
  • Sequential image processing
  • Background job management

🎲 Rarity & Distribution Systems

📊 Bell Curve Distribution (Recommended)

Creates natural rarity distribution where:

  • Common traits appear frequently
  • Rare traits appear occasionally
  • Very rare traits appear seldom
  • Mimics real-world trait distribution

🎯 Random Distribution

Pure random selection where:

  • All traits have equal chance
  • No rarity weighting
  • Good for experimental collections

⚖️ Equal Distribution

Balanced distribution ensuring:

  • Each trait appears roughly the same number of times
  • Perfect for limited trait sets
  • Predictable rarity structure

⚙️ Advanced Layer Configuration

🎛️ Layer Toggles & Probabilities

Fine-tune your collection with precise controls:

  • Toggle On/Off: Enable or disable entire layers
  • Probability %: Set appearance chance (1-100%)
  • Real-time Preview: See layer structure instantly
  • Smart Defaults: Backgrounds at 100%, accessories at lower %

📈 Usage Examples

Standard Collection:

  • Background: 100% (every NFT gets a background)
  • Body: 100% (every NFT gets a body)
  • Accessories: 30% (only 30% of NFTs get accessories)
  • Rare Items: 5% (very rare special items)

🏷️ Collection Configuration

📝 Metadata Settings

  • Collection Name: Appears in NFT metadata
  • Description: Detailed collection description
  • Custom IPFS CID: For custom image hosting
  • Collection Size: 1-10,000 NFTs supported

📦 Output Structure

Generated collections include:

  • images/ - All generated NFT images
  • metadata/ - JSON metadata for each NFT
  • collection.json - Collection overview
  • rarity.json - Trait rarity analysis

⚡ Performance & Technical Limits

🚀 Generation Performance

  • Small Collections (1-100): ~30 seconds
  • Medium Collections (100-1000): 2-5 minutes
  • Large Collections (1000-5000): 5-15 minutes
  • Very Large (5000-10000): 15-30 minutes

📊 System Requirements

  • File Size: Up to 10MB per image
  • Total Files: Up to 10,000 files per upload
  • Image Format: PNG with transparency
  • Resolution: Auto-compressed to 1024x1024

💡 Best Practices & Tips

🎨 Design Guidelines

  • Use consistent canvas sizes (1024x1024 recommended)
  • Include transparent backgrounds for overlays
  • Test layer combinations before generating
  • Use descriptive filenames for better organization

⚙️ Optimization Tips

  • Start with small collections to test
  • Use Bell Curve rarity for natural distribution
  • Set appropriate layer probabilities
  • Monitor generation progress in real-time

🔧 Troubleshooting

  • Large collections may take several minutes
  • Ensure all images are valid PNG files
  • Check layer folder structure is correct
  • Use browser refresh if upload fails
